TextField

Ein TextField ermöglicht es dem Nutzer, Text über die Tastatur in ein User-Interface einzugeben. Ein erforderliches Label und eine optionale FieldDescription unterstützen dabei, die Anforderungen an die Eingabe klar zu kommunizieren.GitHub

Playground

Verwende <TextField />, um ein Textfeld anzuzeigen. Das TextField muss immer um ein <Label /> erweitert werden. Das Label vermittelt Informationen die für das Verständnis der Eingabeanforderungen unerlässlich sind. Optional kann auch eine <FieldDescription /> ergänzt werden, um zusätzliche Informationen bereitzustellen.

Beginnt mit https://
import Label from "@mittwald/flow-react-components/Label";
import TextField from "@mittwald/flow-react-components/TextField";
import FieldDescription from "@mittwald/flow-react-components/FieldDescription";

<TextField>
  <Label>URL</Label>
  <FieldDescription>Beginnt mit https://</FieldDescription>
</TextField>;

Value

Standardmäßig ist der Value eines TextFields leer. Es ist möglich dem TextField eine defaultValue mitzugeben.

import Label from "@mittwald/flow-react-components/Label";
import TextField from "@mittwald/flow-react-components/TextField";

<TextField defaultValue="https://mittwald.de">
  <Label>URL</Label>
</TextField>;

Es kann auch ein value gesetzt werden, um das TextField controlled zu verwenden, und über onChange zu updaten.

Input Properties

Das TextField unterstützt die Standard-Properties eines HTML <input /> Feldes. (s. MDN)

import Label from "@mittwald/flow-react-components/Label";
import TextField from "@mittwald/flow-react-components/TextField";

<TextField name="email" type="email">
  <Label>E-Mail</Label>
</TextField>;

States

Disabled

Wenn das TextField disabled ist, kann nicht mit ihm interagiert werden, was auch optisch verdeutlicht wird.

Validierung

Wenn die Benutzereingabe nicht den erwarteten Anforderungen entspricht, wird das TextField invalidiert. Informationen zum Fehler wird im <FieldError /> angezeigt. Das TextField unterstützt zur Validierung

  • isRequired für Pflichtfelder
  • validate für eigene Validierungen
  • HTML input Properties wie type="email", minLength
Das ist keine URL

TextFields ermöglichen es Nutzern, freien Text über das Keyboard einzugeben. Ein erforderliches Label und eine optionale FieldDescription unterstützen dabei, die Anforderungen an die Eingabe klar zu kommunizieren oder Fehlermeldungen darzustellen.

Best Practices

  • Frage nur nach Informationen, die wirklich notwendig sind.

  • Stelle sicher, dass das Label prägnant, vollständig sichtbar und leicht verständlich ist.

  • Verwende die FieldDescription sparsam und für sinnvolle Beispiele oder Formatierungshinweise.

  • Stelle sicher, dass Fehlermeldungen hilfreich sind, eine Lösung bieten und leicht umzusetzen sind.

  • Verwende keine Placeholder für wesentliche Informationen.

Verwendung

Position

Das TextField wird meistens innerhalb eines ColumnLayouts platziert. Es füllt dabei jeweils die Breite des Columns, in dem es platziert wird, vollständig aus. Dadurch folgt es auch einer vorhandenen Aufteilung in Spalten, die das ColumnLayout eventuell mit sich bringt. Das ColumnLayout macht es somit sehr einfach, übersichtliche Forms unter Berücksichtigung des Layouts zu erstellen.

Am besten eignet sich ein 1:1 Raster für TextFields, sodass sie maximal die Hälfte des Inhaltsbereichs einnehmen können. Das Column Layout kann außerdem erweitert werden, um spezifischeren Layoutanforderungen gerecht zu werden. So kann bspw. ein 2:1 Raster innerhalb einer Spalte des 1:1 Rasters platziert werden, um Straßenname und Hausnummer zu einer sinnvollen Gruppe zusammenzufassen.

Grids und sinnvolle Gruppen

Nutze ein 1:1 ColumnLayout für TextFields. Um sinnvolle Gruppen (z.B. Straße und Hausnummer) zu erstellen, kannst du mehrere ColumnLayouts ineinander verschachteln.

isRequired

Um ein TextField als erforderlich zu markieren, benutze das dazugehörige Property isRequired. Felder, die nicht required sind, werden als optional gekennzeichnet.

Inhalt

Label

Jedes TextField muss ein zugehöriges Label besitzen. Das Label ist entscheidend, um die Eingabeanforderungen zu verstehen und sollte daher so gewählt werden, dass es alle Informationen liefert, um das TextField korrekt ausfüllen zu können. Außerdem sollte das Label knapp (max. 2 Wörter), präzise und weder versteckt noch auspunktiert sein.

FieldDescription

Die FieldDescription kann als optionaler Hilfstext unterhalb des TextFields angezeigt werden, um dem Nutzer ergänzenden Informationen oder Anweisungen zu bieten. Es sollte sparsam verwendet werden und kann als zusätzliche Beschreibung verwendet werden, Eingabe- und Formatierungsbeispiele liefern.

Placeholder

Placeholder verursachen verschiedene Accessibility-Probleme. Sie verschwinden beim Beginn der Texteingabe, werden von manchen Assistenztechnologien nicht korrekt erkannt und sind kognitiv schwieriger zu verarbeiten, als bspw. eine FieldDescription.
Daher dürfen Placeholder keine Informationen enthalten, die für das korrekte Ausfüllen des TextFields erforderlich sind.

Ergänze das TextField mit ...

  • Einem Label für Informationen, die essenziell für das Verständnis der Eingabeanforderungen sind.
  • Einer FieldDescription, um dem Nutzer Beispiele oder Formatierungshinweise zu geben.

Dont

Placeholder sollten keine hilfreichen Informationen oder Formatierungshinweise enthalten.

Do

Nutze die FieldDescription um hilfreiche Informationen darzustellen, z.B. für Formatierungshinweise.

Verhalten

Min / Max Width

Ein TextField besitzt keine fixe Breite. Achte daher darauf, die Breite des TextFields angemessen zum Inhalt zu wählen. So kann z.B. für ein TextField für eine URL deutlich mehr Platz benötigt werden, als für eines, das nur eine Hausnummer abfragt.

Accessibility

Das TextField kann sowohl über Tastaturnavigation erreicht und aktiviert werden als auch von Assistenztechnologien angesteuert werden. Ein User kann außerdem im TextField befindliche Icons auswählen, wenn diese interaktiv sind.

Damit auch der Inhalt barrierefrei ist, solltest du:

  • Essenzielle Eingabeanforderungen über das Label kommunizieren.
  • Unterstützung, Beispiele oder Formatierungshilfen über die FieldDescription bieten.
  • Placeholder nicht für Inhalte verwenden, die zum Ausfüllen des TextFields hilfreich sind.
  • Das Aria-Label pflegen, wenn die Komponente ohne Label verwendet wird.
